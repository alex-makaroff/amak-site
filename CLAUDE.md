## Project Overview

AMAK Sites - a portfolio/agency website for a web design studio. Russian-language content. Monorepo with two parts:
- **Frontend**: React 18 SPA (Vite + Tailwind CSS v4 + TypeScript)
- **Backend/CMS**: Strapi 4.25.9 (Node.js, PostgreSQL)

In production, Strapi serves both the API and the built frontend as static files via SPA fallback middleware.

## Commands

### Setup
```bash
npm install                  # Frontend dependencies
npm run setup:cms            # CMS dependencies (uses --legacy-peer-deps)
cp .env.example .env         # Configure environment variables
```

### Development
```bash
npm run dev                  # Vite dev server (port 5173, proxies /api, /uploads, /admin to CMS on PORT from .env)
npm run dev:cms              # Strapi dev server (PORT from .env, reads root .env)
```

### Build & Production
```bash
npm run build                # Full build: Vite → copy dist to cms/public → install CMS deps → Strapi build
npm start                    # Production: Strapi on PORT from .env, serves frontend + API
```

### Deployment
```bash
node update.js [-b branch] [-f]                                    # Auto-update (git pull, rebuild, restart)
node deploy/srv.cjs install|delete|reinstall [-n name] [-v version] # Systemd service management
```

## Architecture

### Frontend (`src/`)
- **Entry**: `src/main.tsx` → `src/app/App.tsx`
- **Router**: react-router v7 with routes: `/`, `/cases`, `/cases/:slug`
- **Components**: `src/app/components/` - page sections (Hero, Cases, FAQ, etc.)
- **UI Kit**: `src/app/components/ui/` - shadcn/ui components (Radix + Tailwind)
- **Figma exports**: `src/imports/` - auto-generated reference components (not used in app)
- **API client**: `src/app/lib/strapi.ts` - typed Strapi API helpers
- **Styles**: Tailwind v4 (`src/styles/tailwind.css`), custom theme (`src/styles/theme.css`)
- **Path alias**: `@` maps to `./src` (configured in `tsconfig.json` and `vite.config.ts`)

### CMS (`cms/`)
- **Strapi 4** with PostgreSQL (`pg`), has its own `package.json` (required by Strapi)
- **Content types**: `case` (collection), `homepage` (single), `cases-page` (single)
- **Components**: `case.website-details`, `case.design-details`, `case.design-screen` (dynamic zone)
- **Custom API**: `cms/src/api/contact/` - sends contact form submissions to Telegram bot
- **Middlewares**:
  - `spa-fallback.js` - serves `index.html` for non-API routes (SPA routing)
  - `cloud-uploads.js` - proxies `/uploads/` from Replit object storage
- **Bootstrap** (`cms/src/index.js`): auto-creates admin user and sets public API permissions on first run

### Scripts (`scripts/`)
- `strapi.js` - cross-platform Strapi runner, loads root `.env` and runs Strapi commands from `cms/`
- `copy-frontend.js` - copies Vite build output (`dist/`) to `cms/public/` (cross-platform replacement for shell script)
- `copy-frontend.sh` - legacy bash version (kept for Linux deployments)

### Build Pipeline
Vite builds to `dist/` → `scripts/copy-frontend.js` copies output to `cms/public/` → Strapi build. In production, Strapi serves everything on the port defined by `PORT` in `.env`.

### Environment
Single `.env` file in project root (see `.env.example`):
- **Server**: `HOST`, `PORT`, `NODE_ENV`
- **Strapi keys**: `APP_KEYS`, `API_TOKEN_SALT`, `ADMIN_JWT_SECRET`, `JWT_SECRET`
- **Database**: `DATABASE_URL` (PostgreSQL connection string)
- **Telegram**: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` (contact form notifications)
- **Frontend**: `VITE_API_BASE_URL` (optional, defaults to same origin)

The `scripts/strapi.js` loader reads root `.env` before starting Strapi, so `cms/.env` is not needed.
Vite dev server reads `PORT` from `.env` to configure proxy target automatically.

## Key Conventions
- The API/CMS port is configured via `PORT` in `.env` — never hardcode it
- Case types are `"website"` or `"design"` - determines detail component rendering
- CMS dependencies require `--legacy-peer-deps` flag
- `src/imports/` contains Figma-generated reference components - actual app components are in `src/app/components/`
- `cms/build/` is gitignored - regenerated by `npx strapi build`
- `vite.config.js` is gitignored - only `vite.config.ts` is the source of truth
